## Improvements

* The Spark version has been updated to 3.2.1. See [Spark Overview](https://spark.apache.org/docs/latest/) for more information.

## Bug Fixes

* Fixed an issue that caused the [**click_signals_aggregation**](/fusion/5.5/590/aggregation-jobs) job to fail with a "Driver JVM" error.

* Fixed an issue that caused containers to crash when an Argo-based job failed to list the job status.

## Known Issues

* A bug was discovered in the Spark 3.2.x component that prevents signal aggregation for signals stored in the cloud in Parquet format. For more information about the issue, see the [Spark bug report](https://issues.apache.org/jira/browse/SPARK-38318).

    <Note>
    This issue is fixed in Fusion 5.7.1.
    </Note>

## Removals

<Warning>
The following features and functionalities are removed from default deployment in Fusion 5.5.2. See [Deprecations and Removals](/fusion/5.5/8855/deprecations-and-removals) for more information about suggested replacements:
</Warning>

* [Jupyter](/fusion/5.5/118/jupyter-support-in-fusion)
* [Superset](/how-to/how-to/afsrk7/enable-superset-fusion-5-4-and-up)

## Known issues

* New Kerberos security realms cannot be configured successfully in this version of Fusion.
